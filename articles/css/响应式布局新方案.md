### <font style="color:rgb(60, 112, 198);background-color:rgb(237, 237, 237);">概念：融合响应式设计</font><font style="color:black;background-color:rgb(237, 237, 237);">  
</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">本文中我们可以尝试得出一种新的解决方案，那就是利用 JavaScript 和 CSS 来进行媒体查询，是响应式设计与自适应设计结合的方案，我这里给起一个好记的名字叫做融合响应式设计（ Fusion Web Design，简称 FWD ）。</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">技术原则：</font>

+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">JavaScript 判断尽量少</font>
+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">CSS 媒体查询的断点也尽量少</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">UI效果</font>

+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">在笔记本、平板电脑、智能手机上展示合适的 UI 效果</font>
+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">PC UI：给笔记本等设备上看的大屏效果</font>
+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">Mobile UI：给智能手机上看的小屏效果</font>

### <font style="color:rgb(60, 112, 198);background-color:rgb(237, 237, 237);">先说为何不能以 UserAgent 来作为依据</font>

#### <font style="color:black;background-color:rgb(237, 237, 237);">源码示例</font>
```jsx
import React from 'react' 
// JavaScript 的媒体查询 
const mqlMedia = window.matchMedia('(orientation: portrait)')

function onMatchMediaChange(mql = window.matchMedia('(orientation: portrait)')) { 
  if (mql.matches) { 
    //竖屏 
    return 'portrait' 
  } else { 
    //横屏 
    return 'horizontal' 
  } 
} 
// 输出当前屏幕模式 
const getUiMode = (uiMode = '', mql) => { 
  if (uiMode) return uiMode 
  if (!('onorientationchange' in window)) return 'pc' 
  let status = onMatchMediaChange(mql) 
  let width = status === 'portrait' ? Math.min(window.innerWidth, window.innerHeight) : Math.max(window.innerWidth, window.innerHeight) 
  if (width > 1040) return 'pc' 
  return 'mobile' 
} 
const getIsPcMode = (uiMode) => uiMode === 'pc' 
/** 
 * UI 模式，判断逻辑 
 * @export 
 * @param {*} Cmp 
 * @returns 
 */ 
export function withUiMode(Cmp, options = {}) { 
  return class WithUIRem extends React.Component { 
    constructor(props) { 
      super(props) 
      let uiMode = getUiMode() 
      let isPCMode = getIsPcMode(uiMode) 
      this.state = { 
        uiMode: uiMode, 
        isPCMode: isPCMode, 
      } 
    } 
    // 横竖屏切换监听 
    componentDidMount() { 
      mqlMedia.addListener(this.changeUiMode) 
    } 
    componentWillUnmount() { 
      mqlMedia.removeListener(this.changeUiMode) 
    } 
    changeUiMode = (mql) => { 
      let newUiMode = getUiMode('', mql) 
      if (newUiMode !== this.state.uiMode) { 
        this.setState({ 
          isPCMode: getIsPcMode(newUiMode), 
          uiMode: newUiMode 
        }) 
      } 
    } 
    render() { 
      return <Cmp {...this.state} {...this.props} /> 
    } 
  } 
} 
export default (options) => { 
  return (Cmp) =>  withUiMode(Cmp, options) 
}
```

### <font style="color:rgb(60, 112, 198);background-color:rgb(237, 237, 237);">符合设计师需求的细节</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">在区分好 PC UI 和 Mobile UI 的判断设定后，我们还要关心的是在一些常见的设备上的显示效果，比如 iPad 横竖屏、大屏显示器。</font>

#### <font style="color:black;background-color:rgb(237, 237, 237);">Mobile UI 效果中的优化</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">在 Mobile UI 模式下，iPad Mini 显示效果如下：</font>

+ <font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">竖屏的为 768px，此时显示的平铺拉伸版效果</font>
+ <font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">横屏的为 1024px，此时显示的平铺拉伸版效果</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">所以 Mobile UI 就有了如下的 CSS 最大宽的设定：</font>

```jsx

// Mobile 模式下页面最大宽 
$max-body-width = 768px 
// Mobile 模式下的内容宽度 
$max-body-width-center = $max-body-width - 40px
```

#### <font style="color:black;background-color:rgb(237, 237, 237);">PC UI 效果中的优化</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">在上面的 PC UI 的判定中，我们是以 1040px 作为判断区间的，那么在这个宽度下，推荐设计师以 960px 为设计宽度，这样两侧就各有 40px 的留白，以此来增加整个页面的呼吸感。</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">那么，市面上绝大部分的设备其实是比 1280px 还要大的。此时，可以选择以 1220px 作为更大屏的媒体查询断点。也就是说，我这里运用了响应式设计中的 CSS 媒体查询。</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">为何不选用 1400px 呢，因为在 CCtalk 上课的网师、学生所使用的主流设备分辨率宽度还是在 1280px、1366px、1440px这几个区间的。</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">所以，有如下 PC UI 的 CSS 最大宽设定：</font>

```jsx
// PC 模式下小屏的宽度断点 
$page-min-width = 1040px 
// 小屏模式下的内容宽度 
$page-min-width-center = $page-min-width - 80px 
// PC 模式下大屏的宽度断点 
$page-max-width = 1220px // 触屏三合一上的最大宽 
// 大屏模式下的内容宽度
```

### <font style="color:rgb(60, 112, 198);background-color:rgb(237, 237, 237);">设计规范</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">使用融合响应式设计时，需要尽量在设计师进行 UI 设计前就与其商量好设计规范。</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">我认为需要注意如下两点：</font>

1. <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">只修改模块的宽度、边距</font>
2. <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">保证模块内的所有细节都不调整模块细节内容的 UI，如字体、宽高、间距</font>

<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">如果 Mobile UI 与 PC UI 的细节差异很大，那无论是响应式设计、自适应设计，还是我认为的融合响应式设计，在网页制作过程中，都是非常难做的，后期维护成本也是非常高的。</font>

### <font style="color:rgb(60, 112, 198);background-color:rgb(237, 237, 237);">小结</font>
<font style="color:rgb(74, 74, 74);background-color:rgb(237, 237, 237);">融合响应式设计（ Fusion Web Design，简称 FWD），利用 JavaScript 和 CSS 来进行媒体查询，是响应式设计与自适应设计结合的方案。</font>

+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">自适应设计：根据设备特点，使用 JavaScript 来区分 PC UI 和 Mobile UI</font>
+ <font style="color:rgb(1, 1, 1);background-color:rgb(237, 237, 237);">响应式设计：在需要调整出更好的 UI 时，可以加入一些 CSS 媒体查询的断点，比如常见的大屏显示效果</font>
