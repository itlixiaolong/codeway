# Vue2 与 Vue3 Diff 算法区别（面试版）

本文档适配 VitePress 渲染规范，聚焦 Vue2 与 Vue3 Diff 算法的核心差异，包含面试高频考点、算法解析、对比表格及标准回答，可直接复制到 VitePress 项目中使用，兼顾可读性与考点完整性。

## 一、核心区别（一句话总结，面试开篇必答）

- **Vue2**：纯运行时双端对比，全量遍历虚拟 DOM，无编译期优化，靠启发式算法减少 DOM 操作。

- **Vue3**：编译期标记 \+ 运行时靶向更新 \+ 最长递增子序列（LIS），仅对比动态节点，实现最少 DOM 移动。

## 二、整体设计思路差异

### 2\.1 Vue2 Diff 设计思路

- 完全依赖 **运行时** 进行 Diff 对比，无任何编译期优化操作。

- 每次组件更新，都会 **全量遍历整棵虚拟 DOM 树**，即使是无动态绑定的静态节点，也会参与对比。

- 列表 Diff 核心采用 **双端双指针启发式对比**，尽量减少 DOM 移动。

### 2\.2 Vue3 Diff 设计思路

- 采用 **编译时优化 \+ 运行时优化** 结合的方式，大幅提升更新性能。

- 编译阶段：给虚拟 DOM 节点打上 `PatchFlag` 标记，区分静态节点与动态节点，明确动态节点的更新类型。

- 运行阶段：仅对带有 `PatchFlag` 的 **动态节点** 进行 Diff 对比，直接跳过静态节点。

- 列表 Diff 采用 **前后预处理 \+ 最长递增子序列（LIS）** 算法，确保 DOM 移动次数最少。

## 三、列表 Diff 核心算法对比（面试重点）

### 3\.1 Vue2：双端对比算法

核心逻辑：通过四个指针控制对比范围，依次尝试匹配，减少 DOM 操作，属于启发式算法（非最优解）。

#### 核心指针

- `oldStartIndex`：旧列表起始索引

- `oldEndIndex`：旧列表结束索引

- `newStartIndex`：新列表起始索引

- `newEndIndex`：新列表结束索引

#### 匹配顺序

1. 头头匹配：旧列表起始节点 ↔ 新列表起始节点

2. 尾尾匹配：旧列表结束节点 ↔ 新列表结束节点

3. 旧头新尾匹配：旧列表起始节点 ↔ 新列表结束节点

4. 旧尾新头匹配：旧列表结束节点 ↔ 新列表起始节点

若以上四种匹配均失败，则遍历旧列表，根据 `key` 查找可复用节点。

#### 算法缺点

- 属于启发式算法，DOM 移动次数不一定是最优解。

- 最坏时间复杂度接近 O\(n²\)（当列表完全逆序时）。

- 无法精准定位最少移动次数，性能上限较低。

### 3\.2 Vue3：快速前后比对 \+ LIS 算法

核心逻辑：先通过前后预处理快速缩小对比范围，再对中间乱序部分用 LIS 算法，实现 DOM 移动次数最少（最优解）。

#### 核心步骤

1. 前后预处理：从头、从尾同步比对新旧列表，跳过内容相同的节点，快速缩小中间乱序片段的范围。

2. 乱序片段处理：对中间乱序的节点，建立 `key → 新列表索引` 的映射关系。

3. LIS 计算：找出乱序片段中「不需要移动的最长递增子序列」，该序列的节点可直接复用，无需移动。

4. 节点移动：仅对最长递增子序列外的节点进行移动，确保 DOM 操作次数最少。

#### 算法复杂度

- 前后预处理：O\(n\)（线性遍历）

- LIS 计算：O\(n log n\)（最优时间复杂度）

- 整体效率远高于 Vue2 的双端对比算法，尤其适合长列表场景。

## 四、Vue3 独有的编译期 Diff 优化（核心亮点）

Vue3 之所以 Diff 性能优于 Vue2，核心在于新增了**编译期优化**，提前标记动态节点，避免无效对比。

### 4\.1 PatchFlag（靶向更新）

编译时给动态节点打上「更新类型标记」，运行时仅对比标记对应的内容，跳过无关属性/节点，实现精准更新。

常用 PatchFlag 类型：

- `TEXT`：仅文本内容变化（如 `\{\{ msg \}\}`）。

- `CLASS`：仅 class 绑定变化（如 `:class=\&\#34;activeClass\&\#34;`）。

- `STYLE`：仅 style 绑定变化（如`:style=\&\#34;\{ color: red \}\&\#34;`）。

- `PROPS`：指定属性变化（如 `:id=\&\#34;item\.id\&\#34;`）。

- `FULL\_PROPS`：全量属性对比（无明确指定动态属性时）。

### 4\.2 静态提升（Static Hoisting）

纯静态节点（无任何动态绑定、无事件、无插槽）会被提升到渲染函数之外，**只创建一次**，后续组件更新时，直接跳过该节点的 Diff 对比，大幅减少冗余操作。

### 4\.3 Block Tree（块树优化）

编译时将页面中的动态节点收集到「Block 块」中，形成 Block Tree。更新时，仅遍历 Block 块内的动态节点，不再递归整棵虚拟 DOM 树，减少遍历范围。

### 4\.4 事件缓存

内联事件函数（如 `@click=\&\#34;handleClick\&\#34;`）会被缓存，避免每次组件渲染时重新生成函数，减少不必要的 Diff 对比和函数创建开销。

## 五、面试高频对比表（清晰易记）

|对比维度|Vue2|Vue3|
|---|---|---|
|核心 Diff 算法|双端双指针对比（启发式）|前后预处理 \+ 最长递增子序列（LIS）|
|对比范围|整棵虚拟 DOM 全量对比|仅对比带 PatchFlag 的动态节点|
|静态节点处理|每次更新都参与 Diff 对比|静态提升，完全跳过 Diff|

|编译期优化|无任何编译期优化|PatchFlag、静态提升、Block Tree、事件缓存|
|更新粒度|组件级更新为主|元素级、属性级精准更新|
|时间复杂度|近似 O\(n\)，最坏 O\(n²\)|O\(n log n\)（LIS 主导）|

## 六、面试标准回答（可直接背诵）

Vue2 和 Vue3 的 Diff 算法核心区别主要有三点，精准对应考点，简洁不冗余：

1. 算法不同：Vue2 采用双端双指针对比（启发式），Vue3 采用前后预处理 \+ 最长递增子序列（LIS），实现最少 DOM 移动。

2. 优化时机不同：Vue2 只有运行时优化，无编译期操作；Vue3 新增编译期优化（PatchFlag、静态提升等），提前标记动态节点。

3. 对比范围不同：Vue2 全量遍历整棵虚拟 DOM，Vue3 只对比带标记的动态节点，性能大幅提升。

## 七、实际开发注意事项（延伸考点）

- 列表渲染必须使用**稳定、唯一的 key**，避免使用 index 作为 key，否则会导致 Diff 优化失效，甚至出现 DOM 渲染异常。

- Vue3 中，大块静态内容（如固定文本、无动态绑定的布局）会被自动静态提升，无需手动优化。

- Vue3 动态绑定越具体（如明确绑定单个属性），PatchFlag 标记越精准，Diff 效率越高。
